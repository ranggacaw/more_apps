Admin Operations

Admin Back Office

Admin workflows coordinate clinic operations.

Admin routes are limited to verified `admin` users. The module spans operational dashboards, assisted booking, live queue management, internal payment collection, communications, content publishing, user provisioning, scheduling, master data, and management reports.

dashboard

Operational Overview

Recent bookings, payments, queue counts, patient context, and pending internal treatment handoffs.

booking

Assisted Booking

Creates confirmed standard or outside-hours bookings for registered or guest patients without Midtrans.

queue

Walk-In Queue

Tracks waiting, assigned, and in-consultation walk-ins with polling APIs for admin and doctor screens.

billing

Invoices

Finalizes internal package invoices and consultation-treatment handoffs after on-site collection.

comms

Broadcasts and Content

Queues WhatsApp broadcasts and publishes educational content surfaced on the homepage.

setup

Users and Master Data

Manages operational accounts, doctor profiles, clinic hours, aesthetic programs, and reports.

Booking Creation

Admin-assisted bookings confirm immediately.

Admins book registered or guest patients into active doctor slots. Online bookings request a doctor-provided Google Meet link and all successful bookings dispatch notifications.

Assisted Booking Request Lifecycle

The flow bypasses Midtrans entirely and does not award consultation credits.

Clinic Hours

Operating hours drive slot search and override audits.

Admins manage active `clinic_operating_hours`; bookings outside those windows require explicit override intent and reason, stored in `schedule_override_logs`.

Collection

Internal billing is finalized by admins.

Doctor completion can create pending internal package invoices and treatment handoff payments. Package invoices activate entitlements; treatment handoffs only record collected revenue.

Package Invoice and Treatment Handoff Finalization

Both paths lock the payment row and reject already processed records.

Communications

Broadcast delivery always runs asynchronously.

Admins create approved-scope WhatsApp broadcasts. Recipient delivery rows are inserted immediately, and the job processes provider calls with per-delivery error capture.

WhatsApp Broadcast Lifecycle

Audience scopes are restricted to doctors, admins, and all operational users.

Provisioning

Admin users page manages operational accounts.

The user module provisions doctors, admins, and super admins, can mark accounts verified directly, and keeps doctor profiles historically preserved when roles change.

Doctor profiles are created or updated for doctor roles and deactivated when a doctor becomes another role.

Master Data

Aesthetic programs and reports complete the admin back office.

Admins maintain active Aesthetic Program options used by doctors and run cash-basis reports for management visibility.
