System Architecture

Architecture and Data

Staff-operated clinic platform boundaries.

The active application is centered on verified operational users. Admins create bookings and manage back-office data, doctors complete consultations and program reviews, super admins handle finance mutations, and external providers are reached through queued jobs or webhook callbacks.

roles

Staff Gate

`auth`, `verified`, and role middleware gate dashboards. `doctor`, `admin`, and `super_admin` are operational roles; registered patients use `role = null` where legacy flows still reference users.

inertia

Laravel and React

Laravel controllers assemble Inertia page payloads. React pages render dashboards, tables, workspaces, and polling UI while mutations post back to named routes.

postgres

Central Ledger

`payments` is the billing ledger for consultation, package, and treatment handoff records. Finance reports read paid rows on a cash basis.

clinic

Consultation Sources

Consultations originate from confirmed scheduled bookings or walk-in queue entries, then converge into consultation, line-item, payment, and medical-record data.

async

Queued Messaging

Booking, package, broadcast, reminder, and review notifications dispatch jobs that call email and WhatsApp services outside the request lifecycle.

history

Historical Preservation

Active flags hide future choices without destroying historical links. Consultation line items snapshot names, prices, HPP, and metadata.

Supporting Routes

Auth, profile, and user-guide routes support staff operations.

The app keeps standard Laravel auth flows for login, password reset, email verification, password confirmation, password update, logout, and profile management. The user guide is available only to authenticated verified users.

Scheduling

Slots move from availability to confirmed booking.

Clinic operating hours generate and filter reservable slots. Admin-assisted bookings confirm immediately; retained Midtrans self-service logic still models pending bookings and locked slots.

Slot and Booking Lifecycle

Locks expire through the scheduler, while admin-assisted booking bypasses Midtrans and books the slot immediately.

Walk-In Queue

Walk-ins converge into the same consultation model.

Queue entries capture walk-in identity and complaint data without requiring a user account. Admin assigns a doctor, then the doctor starts and completes the consultation workspace.

Entitlements

Packages activate credits and weekly progress tracking.

Paid package payments activate `user_packages`. Operational package check-ins decrement credits, while weekly program progress check-ins reuse `check_ins` without consuming credits.

Preservation Rules

Active choices change without breaking history.

The system avoids deleting historical billing and consultation context. Master data can be deactivated or soft-deleted while existing payments, line items, and package links retain snapshots.

Historical Snapshot Boundaries

Master data changes only affect future selection; historical rows keep links or payload snapshots.
