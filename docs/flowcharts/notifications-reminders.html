Docs/Overview

Notifications

Async Messaging

Notifications leave requests through database queue jobs.

Booking, package, program, reminder, and broadcast messages are dispatched as jobs. Provider services then send email or WhatsApp messages, gracefully skipping missing contact channels and recording broadcast delivery outcomes.

bookings

Booking Notifications

Confirmation, admin booking, doctor link request, meeting-link-ready, reminders, and completion follow-up messages.

packages

Package Notifications

Activation, weekly check-in reminder, and weekly review available messages for active patient programs.

broadcasts

Broadcast Delivery

Admin WhatsApp broadcasts materialize recipient rows and process them asynchronously.

scheduler

Scheduled Work

Every-minute slot lock release, day-before booking reminders, same-day reminders, and weekly check-in reminders.

providers

Provider Fallbacks

WhatsApp can use log, Fonnte, or Wablas providers; email can log or send raw Laravel mail.

assets

Temporary Links

Clinic assets are stored on the configured disk and surfaced through signed temporary URLs.

Booking Messages

Booking jobs fan out to patients, guests, and doctors.

Booking events dispatch a job with a type. The job reloads the booking, builds a message for the target audience, and sends through available contact channels.

Booking Notification Job Flow

Guest patients usually receive WhatsApp-only messages because there is no registered email account.

Admin Broadcast

Broadcast jobs record per-recipient delivery status.

Admin broadcasts are never sent inline. They create a parent broadcast row plus delivery rows, then a job sends each message and stores success or failure.

Broadcast Job Processing

Provider errors are caught per delivery, so one failure does not stop the entire broadcast.

Providers

Provider services handle channel-specific send behavior.

WhatsApp and email services isolate external provider concerns. Blank contact data is skipped, missing provider tokens log and skip, and broadcast exceptions are recorded per delivery.

Provider Send Boundary

WhatsApp supports log, Fonnte, and Wablas providers; email supports log or Laravel raw mail.

Assets

Clinic assets are stored centrally and surfaced with temporary URLs.

Doctor avatars, content assets, meal-plan PDFs, progress photos, and supporting documents use the configured clinic asset disk. Controllers convert paths into signed temporary URLs for reading.

Clinic Asset Access Flow

`/clinic-assets/{path}` is signed and relative, while controllers usually generate expiring URLs before rendering pages.
