# Change: Refactor to Staff-Only Workflow

## Why
The clinic needs to shift to a staff-driven operation where patients no longer use the application. The doctor now manages the complete consultation workflow in-room, including selecting packages and filling out detailed monitoring forms for Slimming Programs.

## What Changes
- **BREAKING**: Remove Patient role, authentication, self-registration, and OTP verification workflows.
- **BREAKING**: Remove Patient workspace navigation and self-serve package checkout.
- Modify the Doctor Consultation Completion workflow to require/allow the doctor to select a package, which creates an internal billing handoff (invoice) for the Admin.
- Modify the Consultation Completion for Slimming Programs to capture detailed metrics (Weight, BMI, VFA, Body Fat %, Muscle Mass, Body Measurements, Metabolism/BMR, Anti-Oxidant) instead of general notes.
- Modify Admin workflow to process and finalize package payments based on the invoices generated by Doctors.

## Impact
- Affected specs: `clinic-patient-authentication`, `clinic-access-control`, `clinic-role-workspace-navigation`, `clinic-consultation-delivery`, `clinic-package-commerce`
- Affected code: Role definitions, Patient auth controllers, Navigation layouts, Doctor consultation completion UI/Controller, Admin payment processing, User/Package migrations and models.
